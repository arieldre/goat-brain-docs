How Layers Connect — Per Session

→
Session starts — L0 handoff.md read, "Resuming from: {next action}" announced. L3 + triggered L5 skills always loaded. L4 top-4 Pinecone hits injected into context automatically.
→
During session — L2 loaded on keyword match from BEST_PRACTICES.md header. L5 injected when trigger words appear in user message.
→
Session ends — L0 handoff.md written. L1 AGENTS-LOG appended. Promotion protocol runs: scan handoff → trigger test → promote to L2. L4 entity files updated if new user/project/feedback facts emerged.
→
If domain hit 5+ BPs this session → consider creating L5 skill file. Current candidates: whatsapp-hermes (3 BPs), fly-io (2 BPs), sveltekit (4 BPs).
→
Quarterly — L2 review: archive BPs with Last hit > 90 days in inactive domains. Check for contradicting BPs → merge with "except when" clause.
